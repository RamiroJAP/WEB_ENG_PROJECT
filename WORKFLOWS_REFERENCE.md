## Workflow Details

### 1. test.yml - Code Quality Check
| Setting | Value |
|---------|-------|
| **Triggers** | PR to main/develop, Push to main/develop |
| **Runtime** | ~2-3 minutes |
| **Blocks Merge** | ✅ YES |
| **Checks** | ESLint, dependencies, package.json validation |
| **Failure Message** | "Lint failed - fix code quality issues" |

### 2. security.yml - Security Scanner
| Setting | Value |
|---------|-------|
| **Triggers** | PR, Push, Daily 2 AM UTC schedule |
| **Runtime** | ~3-5 minutes |
| **Blocks Merge** | ✅ YES |
| **Checks** | Secret detection, npm audit, hardcoded credentials |
| **Failure Message** | "Security scan failed - no secrets allowed" |

### 3. build.yml - Build Validation
| Setting | Value |
|---------|-------|
| **Triggers** | PR to main/develop, Push to main/develop |
| **Runtime** | ~1-2 minutes |
| **Blocks Merge** | ✅ YES |
| **Checks** | npm ci, npm run build, dist/ validation |
| **Failure Message** | "Build failed - fix compilation errors" |

### 4. pr-validation.yml - PR Rules
| Setting | Value |
|---------|-------|
| **Triggers** | PR to main/develop only |
| **Runtime** | ~30 seconds |
| **Blocks Merge** | ✅ YES |
| **Checks** | Branch naming, PR description, breaking changes |
| **Failure Message** | "PR invalid - provide description" |

### 5. deploy.yml - Production Deploy
| Setting | Value |
|---------|-------|
| **Triggers** | Push to main only (after all checks) |
| **Runtime** | ~2-3 minutes |
| **Target** | Firebase Hosting (web-eng-project-d657d) |
| **Requires** | FIREBASE_SERVICE_ACCOUNT_WEB_ENG_PROJECT_D657D secret |
| **Auto-deploys** | ✅ YES (on main push) |

## Exit Codes & Error Scenarios

### Exit Code: 0 ✅
```
All checks passed!
- No linting errors
- No secrets detected
- Build successful
- PR valid
```

### Exit Code: 1 ❌
```
Check failed - merge blocked

Possible causes:
1 - ESLint errors in code
2 - Hardcoded credentials found
3 - Build compilation failed
4 - npm audit found vulnerabilities
5 - PR description missing
6 - Branch naming invalid
```

## Workflow Concurrency

- **Within PR:** Last commit cancels previous workflow runs
- **Between PRs:** Separate queues (no conflicts)
- **Deploy:** Waits for all PR checks to complete
- **Security:** Always runs, cannot be skipped

## Troubleshooting Workflow Failures

### Check Logs
1. Go to repo **Actions** tab
2. Click failed workflow
3. Click failed job
4. Expand step to see error

### Common Errors & Fixes

| Error | Cause | Fix |
|-------|-------|-----|
| ESLint error | Code quality issue | `npm run lint:fix` |
| Build failed | Compilation error | Check `npm run build` locally |
| Secret detected | Hardcoded credential | Remove secret, use `process.env` |
| npm audit fail | Vulnerable package | `npm audit fix` or upgrade package |
| No PR description | Missing description | Add detailed PR description |
